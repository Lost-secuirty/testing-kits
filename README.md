# testing-kits

Pure-Python stdlib testing-harness collection. **Zero runtime dependencies.**

59 harnesses across reliability, security, AI, and pharmacy-domain testing,
each as a single self-contained file with a paired unittest suite and a
built-in `--self-test` mode.

See [`HARNESS_INVENTORY.md`](./HARNESS_INVENTORY.md) for the full catalog,
[`HARNESS_ROADMAP.md`](./HARNESS_ROADMAP.md) for what's next, and
[`CLAUDE.md`](./CLAUDE.md) for the contributor guide (file layout, naming,
port table, how to add a harness).

## Running

```bash
make test          # full suite
make test-fast     # pharmacy only (~3s)
make test-core     # core only
make test-security
make test-ai
make test-pharmacy
make selftest      # every harness --self-test
make report        # writes STATUS.md
make lint          # py_compile + ruff (if installed)
make clean

# Run a single harness's self-test
python harnesses/core/api_test_harness.py --self-test
python harnesses/pharmacy/srs_test_harness.py --self-test
```

## Layout

```
harnesses/
  core/       reliability, correctness, data, perf, observability
  security/   auth, injection, supply chain, app-security
  ai/         LLM eval, agents, prompt safety
  pharmacy/   pharmacy-domain harnesses
tests/        mirrors harnesses/<cat>/ with test_*.py files
experiments/  in-progress harnesses, excluded from `make test`
template/     harness_template.py — scaffold for new harnesses
tools/        generate_report.py, rewrite_imports.py
```
